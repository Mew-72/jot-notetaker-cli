# Jot CLI + TUI Implementation Plan

## Overview
Transform the current basic note CLI into a full-fledged CLI application with a beautiful keyboard-navigable TUI using Textual framework. Keep JSON storage, add tags/edit/delete/pin/export features.

## Current State
- Single file `jot.py` with argparse CLI
- Commands: add, view, search, clear
- JSON storage at `~/.cli_notes.json`
- Uses pydantic + rich

## Architecture

```
jot/
├── pyproject.toml         # Updated deps
├── jot.py                 # Main entry point (thin, delegates to cli.py)
├── cli.py                 # Click CLI commands (add, edit, delete, list, search, export, tui)
├── tui.py                 # Textual App + all screens
├── models.py              # Pydantic Note model + dataclasses
├── storage.py             # JSON load/save/query logic
├── styles.tcss            # Textual CSS styles
├── jot_app.tcss           # App-level styles
└── screens/
    ├── __init__.py
    ├── note_list.py       # Main list screen
    ├── note_detail.py     # Note detail/preview screen
    ├── note_editor.py     # Add/Edit note screen
    └── confirm.py         # Confirmation dialog
```

## Implementation Steps

### Step 1: Restructure Project & Models
**Files: `models.py`, `storage.py`**

Enhance Note model:
```python
class Note(BaseModel):
    content: str
    tags: list[str] = []
    pinned: bool = False
    created_at: datetime
    updated_at: datetime
```

Storage functions:
- `load_notes() -> list[Note]`
- `save_notes(notes: list[Note])`
- `get_note_by_id(note_id: str) -> Note | None`
- `add_note(content: str, tags: list[str]) -> Note`
- `update_note(note_id: str, content: str, tags: list[str])`
- `delete_note(note_id: str)`
- `search_notes(query: str) -> list[Note]`
- `toggle_pin(note_id: str)`
- `export_notes(format: str, output: Path)`

### Step 2: Click CLI
**File: `cli.py`**

Commands (all work without TUI):
- `jot add "content" -t tag1 -t tag2` — Add note (shorthand: `jot "content"`)
- `jot list [--tag TAG] [--pinned] [--search QUERY]` — List notes
- `jot edit ID "new content" -t tag1` — Edit note
- `jot delete ID` — Delete note (with confirm)
- `jot pin ID` — Toggle pin
- `jot tags` — List all tags
- `jot export --format json|markdown --output FILE` — Export notes
- `jot tui` — Launch TUI (default if no args)

### Step 3: TUI Main App
**Files: `tui.py`, `styles.tcss`**

Textual App structure:
- `JotApp(App)` with header, footer, screen stack
- BINDINGS: q=quit, /=search, n=new, h=home
- Keyboard-only navigation (no mouse)
- Footer shows available key bindings

### Step 4: Note List Screen
**Files: `screens/note_list.py`**

- DataTable showing notes: # | Content | Tags | Pinned | Created
- Navigation: j/k or up/down arrows
- Bindings:
  - `n` — New note (push editor)
  - `Enter` — View note detail
  - `e` — Edit selected note
  - `d` — Delete selected note (confirm modal)
  - `p` — Toggle pin
  - `/` — Focus search bar
  - `Esc` — Clear search
- Search bar at top filters list in real-time
- Pinned notes always shown at top
- Tag filter sidebar (toggled with `t`)

### Step 5: Note Detail Screen
**Files: `screens/note_detail.py`**

- Shows full note content (markdown rendered via Textual Markdown widget)
- Shows tags, created/updated timestamps
- Bindings:
  - `e` — Edit note
  - `d` — Delete note
  - `p` — Toggle pin
  - `Escape` — Back to list

### Step 6: Note Editor Screen
**Files: `screens/note_editor.py`**

- TextArea for note content (multi-line editing)
- Input for tags (comma-separated)
- Bindings:
  - `Ctrl+S` — Save and return
  - `Escape` — Cancel (with confirm if unsaved changes)
- Works for both add and edit (pre-fills for edit)

### Step 7: Confirmation Dialog
**Files: `screens/confirm.py`**

- Modal screen for destructive actions
- Returns True/False via dismiss()

### Step 8: Export Functionality
**File: `cli.py` (export command)**

- JSON: Full note data export
- Markdown: Formatted markdown file with all notes
- Can filter by tag or pinned status

### Step 9: Update Entry Point
**File: `jot.py`**

Thin wrapper: imports and runs click CLI. No logic.

### Step 10: Update pyproject.toml

```toml
dependencies = [
    "click>=8.4.1",
    "pydantic>=2.13.4",
    "rich>=15.0.0",
    "textual>=1.0.0",
]

[project.scripts]
jot = "jot:cli"
```

## Data Migration

- No breaking changes to data format

## Verification Plan
1. **CLI tests**: Run each CLI command manually

   - `jot list` — should show notes
   - `jot list --tag test` — should filter
   - `jot edit <id> "updated"` — should update
   - `jot delete <id>` — should delete with confirm
   - `jot pin <id>` — should toggle pin
   - `jot tags` — should list tags
   - `jot export --format markdown --output notes.md` — should export

2. **TUI tests**: Launch `jot tui` and verify:
   - List displays all notes correctly
   - Keyboard navigation works (arrows, j/k)
   - New note creation flow
   - Edit note flow
   - Delete with confirmation
   - Pin toggle
   - Search filtering
   - Tag filtering
   - Export from TUI (if added)
   - Smooth screen transitions

3. **Edge cases**:
   - Empty note list
   - Notes with special characters
   - Large number of notes (performance)
   - Existing data migration
